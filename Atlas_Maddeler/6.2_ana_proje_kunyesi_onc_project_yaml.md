# 6.2 Ana Proje Künyesi (onc_project.yaml) & Projects Tablosu

> ⚠️ Outdated: 2026-04-10
> Her proje klasörünün kök dizininde bulunacak tekil metadata dosyası tanımlanır. Bu dosya projenin "Dijital Kimliği"dir.
> - **Zorunlu Alanlar:** `uuid` (sistem genelinde tekil), `project_code` (ofis kodu), `project_name`, `client`, `status`, `phase`.
> - **Lokasyon Verisi:** `nas_path`, `server_id`.
> - **Sorumluluklar:** `project_lead_id`, `bim_manager_id`. <input-gerekiyor>

## RDBMS: `Projects` Tablosu Şeması
`onc_project.yaml` dosyasının veritabanı karşılığı olan `Projects` tablosu şu alanları içerir:

| Kolon | Tip | Açıklama |
| :--- | :--- | :--- |
| `uuid` | UUID (PK) | Sistemin tekil kimliği (YAML ile aynı) |
| `project_code` | VARCHAR(20) | Örn: 24001, 24002 (Ofis formatı) |
| `project_name` | TEXT | Projenin tam adı |
| `client_id` | UUID (FK) | `Clients` tablosuna referans |
| `status_id` | INT (FK) | `ProjectStatus` tablosuna referans (Aktif, Arşiv vb.) |
| `phase_id` | INT (FK) | `ProjectPhases` tablosuna referans (Konsept, Ruhsat vb.) |
| `nas_root_path` | TEXT | Proje klasörünün tam NAS yolu |
| `created_at` | TIMESTAMP | Kayıt oluşturma tarihi |
| `extended_props` | JSONB | YAML içindeki diğer tüm opsiyonel alanlar |

### Eşleşme ve Senkronizasyon (Sync)
1.  **Yeni Proje:** Proje açıldığında sistem önce DB'de `uuid` üretir, ardından NAS'ta bu `uuid`'yi içeren `onc_project.yaml` dosyasını oluşturur.
2.  **Güncelleme:** DB'deki kritik değişiklikler (Status, Phase) periyodik olarak NAS'taki YAML'a yazılır (Audit Trail).
3.  **Dinamik Alanlar:** `extended_props` içindeki tüm JSON verisi, raporlama için DB tarafında tam sorgulanabilir haldedir.
